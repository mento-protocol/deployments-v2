## Codebase Patterns
- Addressbook entries follow format: `"Label": "0xAddress"` per network in `.treb/addressbook.json`
- `forge build` exits 1 due to pre-existing `src/Proxy.sol` import error (missing `node_modules/@celo/contracts`) - not related to project code. Use `forge build --skip src/Proxy.sol` for clean builds.
- Remappings go in `remappings.txt` (not `foundry.toml`), Foundry auto-discovers additional remappings from nested `lib/` dirs
- npm install required in `lib/mento-core` for Celo contract imports to resolve
- Working directory is `/home/sol/projects/mento-deployments-v2/.claude/worktrees/feat-wormhole-config-driven`
- NTT contracts have immutable `deployer` check: `msg.sender` during `initialize()` must match `msg.sender` during implementation constructor. CREATE3 breaks this because it uses intermediate contracts. Use NttDeployHelper pattern to work around it.
- Treb's Deployer only supports CREATE2/CREATE3 (no plain CREATE). `sender.execute()` rejects `to=address(0)`.
- Treb uses `vm.prank(sender.account)` for simulation, not `vm.startBroadcast()`. All state changes must go through harness/create3/execute.
- For NTT lookup after deployment: `lookup("NttDeployHelper:USDm")` → read `nttManagerProxy()` and `transceiverProxy()` immutables
- Cross-chain registry lookup: `lookup("NttDeployHelper:USDm", namespace, "143")` where namespace = `vm.envOr("NAMESPACE", string("default"))`
- NttDeployHelper CREATE nonces: nttManagerProxy = nonce 2, transceiverProxy = nonce 4. Use `vm.computeCreateAddress(helper, nonce)` for remote addresses.
- Stack-too-deep in run(): use storage vars populated in setUp() (see SetupNTTBridge / ConfigureNTT pattern)
- WormholeTransceiver.initialize() publishes a Wormhole message requiring `msg.value >= messageFee`. On chains with fee=0 (like Celo), msg.value=0 works.
- `console2.log` supports max 4 total args (format string + 3 values). Split into multiple calls if more args needed.
- NTT proxy upgrades: deploy new impl with SAME constructor args (immutables), call `proxy.upgrade(newImpl)` as owner. `_checkImmutables()` validates consistency.

- **Learnings for future iterations:**

  - Treb's Deployer library ONLY supports CREATE2/CREATE3 (no plain CREATE). sender.execute() rejects to=address(0)
  - Treb uses `vm.prank(sender.account)` for simulation, not vm.startBroadcast()
  - The NttDeployHelper approach: deploy a helper via CREATE3 that does everything in its constructor, preserving msg.sender consistency
  - After deployment, proxy addresses are readable from NttDeployHelper immutables: `nttManagerProxy()`, `transceiverProxy()`
  - Lookup pattern: `lookup("NttDeployHelper:USDm")` → read proxy addresses from the helper
  - NttManager.transferOwnership() cascades to all registered transceivers' ownership

  - WormholeTransceiver.initialize() publishes a Wormhole init message with msg.value — requires messageFee=0 or explicit ETH forwarding
