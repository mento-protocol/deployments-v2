## Codebase Patterns
- Registry lookups: use `lookupOrFail("ContractName:version")` for direct contracts, `lookupProxyOrFail("ContractName")` for proxied contracts
- Proxy types: CELO proxies have prefix `Proxy:`, OZTUP proxies have prefix `TransparentUpgradeableProxy:`
- Config loading: `Config.get()` returns `IMentoConfig`, needs `NETWORK` env var (or `MENTO_CONFIG_CONTRACT`)
- Address lookups: Registry reads both `.treb/registry.json` (uses namespace + chainId) and `.treb/addressbook.json` (uses chainId only)
- Owner pattern: migration owner is the `MigrationMultisig` entry in the addressbook
- TrebScript/ProxyHelper cannot be inherited in tests due to SENDER_CONFIGS env var dependency; use Registry directly instead
- Key V3 identifiers: FPMMFactory (proxy), OracleAdapter (proxy), FactoryRegistry (proxy), VirtualPoolFactory:v3.0.0, Router:v3.0.0, ReserveV2 (proxy), ReserveLiquidityStrategy (proxy), CDPLiquidityStrategy (proxy), BreakerBox:v2.6.5, SortedOracles (proxy), ProxyAdmin, MarketHoursBreaker:v3.0.0, Broker (proxy), ReserveSafe (addressbook)
- Proxy implementation slot (EIP-1967): 0x360894a13ba1a3210667c828492db98dca3e2076cc3735a920a3ca505d382bbc
- Proxy admin slot (EIP-1967): 0xb53127684a568b3173ae13b9f8a6016e243e63b6e8ee1178d6a717850b5d6103
- fs_permissions in foundry.toml already allow reading .treb/registry.json and .treb/addressbook.json
- Implementation addresses in registry use "ContractName:v3.0.0" pattern (e.g. `lookupOrFail("OracleAdapter:v3.0.0")`)
- IFPMMFactory.deployedFPMMAddresses() returns all deployed FPMM pool proxy addresses
- To test init disabled: call initialize() on impl address with vm.expectRevert() — OZ Initializable blocks re-init
